# 单例模式
1. `饿汉模式`
   * 优点：加载阶段即被实例化，线程安全，性能好，实例对象唯一
   * 缺点：长时间不用耗费性能
2. `懒汉模式`
   * 优点：线程安全，性能好
   * 缺点：不保证实例对象唯一性
3. `懒汉模式+synchronized`
   * 优点：线程安全
   * 缺点：不保证实例对象唯一性，synchronized影响性能
4. `DCL（Double Check Locking）`
   * 优点：线程安全，实例对象唯一
   * 缺点：synchronized影响性能
5. `Holder模式`
   * 优点：线程安全，实例对象唯一
   * 缺点：有被反射或反序列或攻击的风险
